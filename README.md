# KW-Pay
## 1. Intro (도입)

- **프로젝트명:** KW Pay (토큰명: KWT)
- **한 줄 소개:** 광운대 학생들의 잃어버린 복지 혜택을 되찾아주는, 캠퍼스 및 지역 상권 상생형 ERC-20 지역 화폐 생태계
- **발표 목적:** 중앙화된 톡학생증 플랫폼에 의존하지 않고, 우리 학교만의 독자적이고 탈중앙화된 결제 및 할인 인프라를 구축하여 학생과 지역 상권 모두가 이득을 보는 모델을 제시한다.

## 2. Problem (문제 제기)

- **톡학생증 혜택의 부재:** 타 대학 학생들은 카카오 톡학생증을 통해 프랜차이즈 카페, 영화관 등에서 다양한 제휴 할인 혜택을 누리고 있다. 하지만 광운대학교는 현재 톡학생증이 발급되지 않아, 광운대학교 학생들은 상대적인 복지 사각지대에 놓여 있다고 볼 수 있다.
- **지역 상권과의 연계 부족:** 다른 학교에 비해 주변 식당과 카페가 많은 편이 아님에도 불구하고 학생회를 통한 산발적인 제휴 외에는 학생들이 상시로 체감할 수 있는 통합된 혜택 관리 시스템이 부재한다.

## 3. Solution (해결 방안)

- **광운대 전용 ERC-20 토큰(KWT) 발행:** 이더리움 기반의 토큰을 발행하여 학교 주변 결제 수단으로 활용하낟.
- **사용자 리워드 및 할인 혜택:** 프랜차이즈 할인을 못 받는 대신, 학생이 제휴 매장에서 KWT로 결제 시 스마트 컨트랙트를 통해 자동으로 10% 할인이 적용되게 하여 토큰 사용성을 극대화한다.

## 4. Stack (스택 상세 설명)

- **블록체인 네트워크:** 이더리움 (또는 가스비가 싼 Polygon 등의 테스트넷)
- **ERC-20 표준:** 이더리움 위에서 나만의 코인을 만들 때 지켜야 하는 규칙. 이 표준을 따랐기 때문에 KWT는 기존의 메타마스크 등 모든 지갑과 상호 운용성을 가진다.
- **개발 언어 (Solidity):** 총발행량을 관리하는 `totalSupply()`, 토큰을 전송하는 `transfer()` 등의 필수 함수를 구현하여 중개자 없이 투명하게 결제와 할인이 작동하도록 만들었다.

## 5. 발전시키고 싶은 부분
- **학생페이지, 제휴매장 페이지 구분:** 로그인 페이지에서 학생, 제휴매장 로그인 페이지를 따로 만들어서 학생회에 직접 컨택이 되지 못한 매장도 본인이 원한다면 언제든지 제휴를 할 수 있도록 한다.
- 현재 학생이 90% 토큰을 내면 제휴 매장도 90% 토큰을 받는걸로 되어있다. 하지만 제휴매장을 100%를 전부 받을 수 있도록 수정하고싶기도 한다. 따라서 중앙거래처?처럼 학생회 예산이나.. 그런걸 통해 10%의 비용을 매장은 그대로 받을 수 있도록 하는 시스템으로 구축해보고싶기도하다. (근데 그러면 제휴가 아닌건가.. 아무튼 더 수정필요)
